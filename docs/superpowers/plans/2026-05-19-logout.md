# Logout Feature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Logout button to the sidebar that clears the session and returns the user to LoginForm, with full MainForm reset on re-login.

**Architecture:** A new `AppApplicationContext` replaces the direct `Application.Run(new MainForm())` call and manages the login→main→logout→login lifecycle. `MainForm` signals logout by setting `Session.IsLoggingOut = true` before closing. `Session.Clear()` resets all state between sessions.

**Tech Stack:** C# 13, .NET 10, WinForms

### Task 2: Introduce AppApplicationContext and rewire Program.cs

**Files:**
- Modify: `Program.cs`

- [ ] **Step 1: Replace Program.cs content**

```csharp
using Dog_Exploder.Forms;

namespace Dog_Exploder;

internal static class Program
{
    [STAThread]
    static void Main()
    {
        ApplicationConfiguration.Initialize();
        Application.Run(new AppApplicationContext());
    }
}

internal sealed class AppApplicationContext : ApplicationContext
{
    public AppApplicationContext() => ShowLogin();

    private void ShowLogin()
    {
        Session.Clear();
        using var login = new LoginForm();
        if (login.ShowDialog() != DialogResult.OK)
        {
            ExitThread();
            return;
        }
        Session.Username = login.Username;
        var main = new MainForm();
        main.FormClosed += OnMainFormClosed;
        MainForm = main;
        main.Show();
    }

    private void OnMainFormClosed(object? sender, FormClosedEventArgs e)
    {
        if (Session.IsLoggingOut)
            ShowLogin();
        else
            ExitThread();
    }
}
```

```csharp
pnlSidebarBottom.Controls.Add(itemSupport);
pnlSidebarBottom.Controls.Add(itemSettings);
```

> **Note on WinForms docking order:** For `DockStyle.Top`, WinForms lays out children from back to front (highest Controls index first). Controls added **last** end up at the **top**. So `itemSettings` (added last) docks to the very top, then `itemSupport`, then `pnlLogoutSeparator`, then `btnLogout` at the bottom.
